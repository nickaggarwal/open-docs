# Site Configuration

The `site.config.json` file is the central configuration file for your documentation site. It controls the navigation structure, tabs, and overall organization of your content.

## Primary Tab

The primary tab is the main documentation section of your site. Configure it in the `primaryTab` section:

```json
{
  "primaryTab": {
    "name": "Documentation"
  }
}
```

## Additional Tabs

You can add additional tabs to organize different sections of your documentation. Configure them in the `tabs` array:

```json
{
  "tabs": [
    {
      "name": "Advanced",
      "url": "advanced"
    },
    {
      "name": "API Reference",
      "url": "api"
    },
    {
      "name": "Changelog",
      "url": "changelog"
    }
  ]
}
```

Each tab requires:
- `name`: The display name of the tab
- `url`: The URL path for the tab (will be used in the route)

## Sidebar Structure

The `navigation` array defines the sidebar structure and content organization. Here's how to configure it:

```json
{
  "navigation": [
    {
      "group": "",
      "pages": ["introduction/introduction"]
    },
    {
      "group": "Getting Started",
      "pages": [
        "getting-started/quickstart",
        "getting-started/sidebar-navigation",
        "getting-started/groups-tabs",
        "getting-started/content-folder",
        "getting-started/customization"
      ]
    },
    {
      "group": "Guides",
      "pages": [
        "guides/creating-pages",
        "guides/image-embeds",
        "guides/markdown-features",
        "guides/code-blocks",
        "guides/tables"
      ]
    }
  ]
}
```

### Navigation Structure Explained

1. **Root Level Pages**
   - Use an empty group (`"group": ""`) for pages that should appear at the root level
   - Example: `["introduction/introduction"]`

2. **Grouped Pages**
   - Create groups to organize related pages together
   - Each group has:
     - `group`: The display name for the group
     - `pages`: Array of page paths relative to the content directory
   - Example:
     ```json
     {
       "group": "Getting Started",
       "pages": [
         "getting-started/quickstart",
         "getting-started/sidebar-navigation"
       ]
     }
     ```

3. **Page Paths**
   - Page paths should be relative to your content directory
   - Format: `folder/file-name` (without the `.mdx` extension)
   - Example: `"guides/creating-pages"`

4. **Optional Icons**
   - Groups can have optional icons using the `favicon` property
   - Example:
     ```json
     {
       "group": "Deployment",
       "favicon": "rocket",
       "pages": ["deployments/aws"]
     }
     ```

## Best Practices

1. **Organize Logically**
   - Group related content together
   - Use clear, descriptive group names
   - Keep the structure shallow (avoid deep nesting)

2. **URL Structure**
   - Use kebab-case for file names (`my-page` instead of `myPage`)
   - Create folders for each major section
   - Keep URLs descriptive but concise

3. **Maintenance**
   - Keep the configuration file organized and well-commented
   - Update the navigation when adding or removing pages
   - Test all links after making changes

## Example Configuration

Here's a complete example of a `site.config.json` navigation structure:

```json
{
  "primaryTab": {
    "name": "Documentation"
  },
  "tabs": [
    {
      "name": "Advanced",
      "url": "advanced"
    },
    {
      "name": "API Reference",
      "url": "api"
    }
  ],
  "navigation": [
    {
      "group": "",
      "pages": ["introduction/introduction"]
    },
    {
      "group": "Getting Started",
      "pages": [
        "getting-started/quickstart",
        "getting-started/configuration"
      ]
    },
    {
      "group": "Deployment",
      "favicon": "rocket",
      "pages": ["deployments/aws"]
    }
  ]
}
```





This structure provides a clean and organized way to handle complex documentation with multiple levels of categorization.
