# Using Code Blocks

Code blocks are essential for technical documentation. Open Docs provides a rich code block experience with syntax highlighting, line numbers, and more.

## Basic Code Block

To create a basic code block, use triple backticks followed by the language identifier:

````md
```javascript
function hello() {
  console.log('Hello, world!');
}
```
````

This will render as:

```javascript
function hello() {
  console.log('Hello, world!');
}
```

- JavaScript/TypeScript (`js`, `ts`, `jsx`, `tsx`)
- HTML/CSS/SCSS (`html`, `css`, `scss`)
- Python (`py`, `python`)
- Go (`go`, `golang`)
- Ruby (`rb`, `ruby`)
- Rust (`rs`, `rust`)
- Shell/Bash (`sh`, `bash`, `shell`)
- JSON/YAML (`json`, `yml`, `yaml`)
- SQL (`sql`)
- C/C++ (`c`, `cpp`, `c++`)
- Java (`java`)
- PHP (`php`)
- Swift (`swift`)
- Kotlin (`kt`, `kotlin`)
- And many more!

## Adding Filenames

You can add a filename to your code blocks by appending `:filename` to the language identifier:

````md
```javascript:app.js
import express from 'express';
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```
````

This will render as:

```javascript:app.js
import express from 'express';
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

## Showcasing Rust Code

Here's how Rust code looks in our new markdown-style code blocks:

```rust:src/main.rs
use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    
    // Insert some values
    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);
    
    // Get a value
    let team_name = String::from("Blue");
    let score = scores.get(&team_name).unwrap_or(&0);
    
    println!("Team {}: {}", team_name, score);
    
    // Iterate over key-value pairs
    for (key, value) in &scores {
        println!("{}: {}", key, value);
    }
}
```

## Go Examples

Go code is also beautifully highlighted:

```go:hello.go
package main

import (
    "fmt"
    "time"
)

func main() {
    // Print welcome message
    fmt.Println("Welcome to the Go playground!")
    
    // Get current time
    currentTime := time.Now()
    fmt.Println("The current time is:", currentTime.Format("15:04:05"))
    
    // Using a channel for concurrency
    messages := make(chan string)
    
    // Start a goroutine
    go func() {
        time.Sleep(time.Second)
        messages <- "Hello from goroutine!"
    }()
    
    // Wait for message
    msg := <-messages
    fmt.Println(msg)
}
```

## Line Numbers

Line numbers are displayed by default. You can toggle them using the button in the top-right corner of the code block.

## Copying Code

Each code block includes a copy button that allows users to easily copy the code to their clipboard.

## Theme Support

Code blocks automatically adapt to the current theme (light or dark mode), ensuring optimal readability in any environment.

## Best Practices

- Always specify the language for proper syntax highlighting
- Use meaningful filenames when showing file-specific code
- Keep code blocks concise and focused on the essential parts
- Consider adding comments to explain complex logic
- For very long code blocks, consider breaking them into smaller, focused examples

By following these guidelines, you can create effective and visually appealing code examples in your documentation.
