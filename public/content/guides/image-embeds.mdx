## Basic Image Embedding

To add an image to your documentation, use the standard Markdown syntax:

```md
![Alt text](path/to/image.png)
```

You can also add a title to provide additional information when users hover over the image:

```md
![Alt text](path/to/image.png "Image title on hover")
```

Example:

![Landscape image](/images/landscape-light.jpg#theme=light "Light mode landscape")
![Landscape image](/images/landscape-dark.jpg#theme=dark "Dark mode landscape")

### Image Sizing

You can control the size of images using HTML attributes:

```md
<img src="path/to/image.png" width="500" height="300" alt="Sized image" />
```

Example:

<img src="/images/landscape-light.jpg#w=500&h=300" alt="Sized image" />

Or for more control, you can use inline styles:

```md
<img src="path/to/image.png" style="width: 500px;" alt="Styled image" />
```

## Theme-Specific Images

You can display different images based on the user's current theme (light or dark mode). This is especially useful for diagrams or screenshots that need different versions for each theme.

### Using Hash Parameters

The simplest way to create theme-specific images is by adding the `theme` parameter to the image URL:

```md
![Alt text](/images/landscape-light.jpg#theme=light "Light mode version")
![Alt text](/images/landscape-dark.jpg#theme=dark "Dark mode version")
```

You can also combine this with other parameters:

```md
![Alt text](/images/landscape-light.jpg#w=400&theme=light "Light mode version")
![Alt text](/images/landscape-dark.jpg#w=400&theme=dark "Dark mode version")
```

### Example Implementation

Here's how the image pair renders (try switching your theme to see the difference):

![Example landscape](/images/landscape-light.jpg#w=500&theme=light "Light mode landscape")
![Example landscape](/images/landscape-dark.jpg#w=500&theme=dark "Dark mode landscape")




## Image Captions

You can add captions to images by wrapping them in a figure element:

```md
<figure>
  <img src="path/to/image.png" alt="Image with caption" />
  <figcaption>This is a caption for the image above</figcaption>
</figure>
```

Example:

<figure>
  <img src="/images/landscape-light.jpg#theme=light" width="500" alt="Landscape with caption" />
  <img src="/images/landscape-dark.jpg#theme=dark" width="500" alt="Landscape with caption" />
  <figcaption>Fig.1 - A beautiful landscape view that adapts to your theme</figcaption>
</figure>

## Clickable Images

To make an image clickable (linking to a larger version or another page), use the following syntax:

```md
[![Alt text](path/to/thumbnail.png)](path/to/full-image.png)
```

Example:

[![Landscape thumbnail](/images/landscape-light.jpg#w=300&theme=light)](/images/landscape-light.jpg)
[![Landscape thumbnail](/images/landscape-dark.jpg#w=300&theme=dark)](/images/landscape-dark.jpg)

Or with HTML for more control:

```md
<a href="path/to/full-image.png" target="_blank">
  <img src="path/to/thumbnail.png" alt="Clickable image" />
</a>
```

### Best Practices for Images

- Create separate optimized versions for light and dark themes
- Ensure both images have the same dimensions to prevent layout shifts
- Use appropriate contrast in each theme version
- Test how images look when the user switches themes
- Always include alt text for accessibility
