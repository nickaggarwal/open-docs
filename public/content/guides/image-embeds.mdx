# Image Embeds

You can embed images in your markdown files using standard markdown syntax.

## Basic Image Embedding

The basic syntax for embedding an image is:

```
![Alt text for the image](/images/landscape.jpg#theme=light "Hover title for the image")
![Alt text for the image](/images/landscape.jpg#theme=dark "Hover title for the image")
```

This will render as:

![Alt text for the image](/images/landscape-light.jpg#theme=light "Hover title for the image")
![Alt text for the image](/images/landscape-dark.jpg#theme=dark "Hover title for the image")

## Image Sizing

You can control the size of images by using HTML attributes, inline styles, or hash parameters.

### Using HTML Attributes

```html
<img src="/images/landscape-light.jpg#theme=light" width="300" alt="Landscape image" />
<img src="/images/landscape-dark.jpg#theme=dark" width="300" alt="Landscape image" />
```

<img src="/images/landscape-light.jpg#theme=light" width="300" alt="Landscape image" />
<img src="/images/landscape-dark.jpg#theme=dark" width="300" alt="Landscape image" />

### Using Inline Styles

```html
<img src="/images/landscape-light.jpg#theme=light" style="width: 50%" alt="Landscape image" />
<img src="/images/landscape-dark.jpg#theme=dark" style="width: 50%" alt="Landscape image" />
```

<img src="/images/landscape-light.jpg#theme=light" style="width: 50%" alt="Landscape image" />
<img src="/images/landscape-dark.jpg#theme=dark" style="width: 50%" alt="Landscape image" />

### Using Hash Parameters for Width

You can also specify the width using hash parameters in the URL:

```
![Alt text](/images/landscape-light.jpg#w=300&theme=light)
![Alt text](/images/landscape-dark.jpg#w=300&theme=dark)
```

![Alt text](/images/landscape-light.jpg#w=300&theme=light)
![Alt text](/images/landscape-dark.jpg#w=300&theme=dark)

## Linked Images

You can make images clickable by wrapping them in a link:

```md
[![Landscape image](/images/landscape-light.jpg#theme=light)](/images/landscape-light.jpg)
[![Landscape image](/images/landscape-dark.jpg#theme=dark)](/images/landscape-dark.jpg)
```

[![Landscape image](/images/landscape-light.jpg#theme=light)](/images/landscape-light.jpg)
[![Landscape image](/images/landscape-dark.jpg#theme=dark)](/images/landscape-dark.jpg)

## Captions with Figure

To add a caption to an image, you can use the `<figure>` and `<figcaption>` elements:

```html
<figure>
  <img src="/images/landscape-light.jpg#theme=light" alt="Landscape" width="500" />
  <img src="/images/landscape-dark.jpg#theme=dark" alt="Landscape" width="500" />
  <figcaption>Fig.1 - A beautiful landscape view (adapts to light/dark mode)</figcaption>
</figure>
```

<figure>
  <img src="/images/landscape-light.jpg#theme=light" alt="Landscape" width="500" />
  <img src="/images/landscape-dark.jpg#theme=dark" alt="Landscape" width="500" />
  <figcaption>Fig.1 - A beautiful landscape view (adapts to light/dark mode)</figcaption>
</figure>

## Using Theme-Specific Images

You can display different images based on the user's theme (light or dark mode) by using hash parameters:

1. Add `#theme=light` to the image URL to show it only in light mode
2. Add `#theme=dark` to the image URL to show it only in dark mode

You can combine multiple parameters like `#w=300&theme=light` for both sizing and theme-specific display.

Try switching your theme to see the difference.

## Best Practices

1. **Always include alt text** for accessibility
2. **Optimize images** before uploading (compress, resize)
3. **Use responsive sizes** when possible
4. **Be consistent** with image styling throughout your documentation
5. **Consider theme-specific images** for better contrast in light and dark modes
